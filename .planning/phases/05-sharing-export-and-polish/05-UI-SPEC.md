## Visual Hierarchy

- **Primary focal point:** the map canvas — sharing surfaces, onboarding overlays, and export controls must never outweigh the map; they are secondary chrome that lives above or beside it
- **Secondary focal point:** the Share button in the top toolbar, and the map title field when active
- **Tertiary focal point:** the share modal, view-mode banner, and onboarding tooltip when visible
- **View mode hierarchy:** in shared read-only view, the map retains full visual dominance; the view-mode banner occupies a narrow strip at the top; the "Edit as Copy" CTA is the only interactive element the viewer sees at first glance
- **Onboarding constraint:** tooltip/overlay must not obscure more than 20% of the map canvas; it is a nudge, not a gate
- **Layer stacking order:** unchanged from Phase 4 for the editor. For the shared view, no editing layers are rendered — the layer stack tops out at station labels. The view-mode banner sits above everything in the document flow (not `position: fixed` on top of the map, but part of the shell layout above the map stage)
- **Export surface:** image export captures only the map canvas region (map stage element), not the toolbar, sidebar, or shell chrome
- **Accessibility rule:** all interactive controls use visible text labels; minimum click target 44x44px for all sharing flow buttons; modal/overlay must trap focus while open; onboarding tooltips must be dismissible with Escape

| Label | 14px | 600 | 1.3 |

Phase 5 type assignments:

- Share modal heading: 20px, weight 600, line-height 1.25 (Heading role)
- Share modal field labels (e.g. "Map Title", "Your name"): 14px, weight 600, line-height 1.3 (Label role)

- Export button label: 14px, weight 600, line-height 1.3 (Label role)

**Share modal surface rules:**

- Modal max-width: 480px, centered horizontally and vertically in the viewport
- Modal background: `var(--shell-dominant)` (`#F3EEE5`)
- Modal header background: `var(--shell-secondary)` — matches toolbar/sidebar chrome
- Modal header text: `var(--shell-dominant)` color
- Modal close button: `×` (U+00D7), 44x44px tap target, top-right corner of the header
- Backdrop: `var(--share-modal-overlay)`, covers full viewport behind the modal
- Modal border-radius: 6px — same rounding convention as existing toolbar buttons

**View-mode banner rules:**

- Banner occupies the top of the shell layout above the map stage (not overlaid on the map)
- Banner height: 48px (matches toolbar height pattern from Phase 1)
- Banner background: `var(--view-mode-banner-bg)`
- Map title: `var(--shell-dominant)` text, 20px weight 600
- Author name (if provided): same line or immediately below title, 14px weight 400, opacity 0.7
- "Edit as Copy" button: right-aligned in the banner, `--shell-accent` background, `var(--shell-dominant)` text, 14px weight 600

**Onboarding tooltip rules:**

- Tooltip is anchored to the target tool/control it describes (appears below the toolbar button it describes)
- Tooltip background: `var(--onboarding-tooltip-bg)`
- Tooltip border: 1px solid `var(--onboarding-tooltip-border)`
- Tooltip border-radius: 4px
- Tooltip padding: `var(--space-sm)` vertical, `var(--space-md)` horizontal
- Max-width: 240px — short nudge copy only
- Arrow/caret: 6px pointing up toward the anchor element, filled with `var(--onboarding-tooltip-bg)`
- A single dismiss button (`Got it`, 14px weight 600, `--shell-accent` color, no background) closes the tooltip
- Multiple tooltips are shown sequentially, not all at once — one tooltip is visible at a time

## Interaction Contract

### Share Button — Toolbar Addition (SHARE-01, SHARE-02, SHARE-03, SHARE-04)

A new `Share` button is added to the top toolbar, positioned to the right of the existing `Add Line` CTA at the far right.

- Label: `Share`
- Style: `--shell-accent` background, `var(--shell-dominant)` text, 14px weight 600, 4px border-radius — same visual treatment as the `Add Line` button
- Behavior: opens the **Share Modal** overlaid above the editor
- The Share button is always present and always interactive, even when the proposal is empty (opening the modal with no lines shows an empty-proposal nudge)

### Share Modal (SHARE-01, SHARE-02, SHARE-03, SHARE-04)

Triggered by: clicking the `Share` button in the toolbar.

Modal layout (top to bottom):

1. **Header strip:** "Share Your Proposal" heading (20px weight 600), close button (`×`, 44x44px) in the top-right corner; header background is `var(--shell-secondary)`, text is `var(--shell-dominant)`
2. **Divider**
3. **Map Title section:**
   - Field label: "Map Title" (14px weight 600)
   - Input: text field, prefilled with `draft.title` (which defaults to "Untitled Proposal" if the user has not set one)
   - The input updates `draft.title` on change — this is a live draft mutation, not a separate share-only field
   - Character limit: 80 characters; show remaining count at 60+ characters as `{N}/80` in 14px muted text
4. **Your Name section (optional):**
   - Field label: "Your name (optional)" (14px weight 600)
   - Input: text field, placeholder `Anonymous`
   - This value is stored with the shared link payload only — it does not mutate the draft
5. **Divider**
6. **Export section:**
   - Subheading: "Export Image" (14px weight 600)
   - Button: `Export PNG` — triggers canvas capture of the map stage element
   - Export button style: `var(--shell-secondary)` background, `var(--shell-dominant)` text, 14px weight 600, full-width button
   - On export start: button label changes to `Exporting...` and is disabled until the file is ready
   - On export complete: file downloads automatically; button returns to `Export PNG`
7. **Divider**
8. **Share Link section:**
   - Subheading: "Share Link" (14px weight 600)
   - Description: "Creates an unlisted link. Anyone with it can view your proposal." (14px weight 400, muted)
   - Button: `Create Link` — persists the current proposal state and generates the unlisted URL
   - Once created, the button area transitions to: a read-only URL field (14px monospace, truncated with ellipsis) + a `Copy Link` button (`--shell-accent` background)
   - After `Copy Link` is clicked: brief `Copied!` feedback replacing the button label for 1.5 seconds, then returns to `Copy Link`
   - If the proposal has no lines: the Share Link section shows a note "Add at least one line before sharing." and the `Create Link` button is disabled (opacity 0.4, non-interactive)
9. **Modal footer:** none — the header close button and clicking the backdrop dismiss the modal

The modal is dismissed by: clicking `×`, pressing `Escape`, or clicking the backdrop overlay.

### Proposal Title Editing (SHARE-01)

The map title is also editable directly in the editor without opening the Share modal. A small `Untitled Proposal` label appears in the toolbar (between the tool group and the baseline toggle), styled as:

- Label text: current `draft.title`, truncated at 24 characters with ellipsis
- Font: 14px weight 400, `var(--shell-dominant)` at 70% opacity when not focused
- On click: the label becomes an inline text input (same font size and weight), `var(--shell-dominant)` text at full opacity, with a 1px `var(--shell-dominant)` bottom border (not a full input box border)
- On blur or Enter: commits the change to `draft.title`; reverts to truncated display label
- Maximum input width: 200px inline

### Shared View Mode (SHARE-05)

When a proposal is loaded via a share link, the app renders in read-only shared view mode. The shared view is structurally the same as the editor but with all editing chrome removed:

- The top toolbar is replaced by the **view-mode banner** (see Color section for banner rules)
- The sidebar is not shown — the full map canvas spans the width
- No drawing tools, no baseline toggle, no corridor toggle, and no comparison toggle are present
- The map can still be panned and zoomed
- Map layers render at full opacity (no comparison mode in shared view)
- The view-mode banner occupies the same 48px height at the top as the toolbar does in the editor
- Banner content (left to right): map title (20px weight 600), author name if set (14px weight 400, muted), [spacer], proposal line count badge (e.g. "3 lines · 12 stations" in 14px weight 600), [spacer], `Edit as Copy` button
- If no author name was set: the author name slot is empty (no placeholder shown)

### Edit as Copy Flow (SHARE-06)

Triggered by: clicking `Edit as Copy` in the view-mode banner.

1. The proposal state is duplicated client-side as a new `ProposalDraft` with a new `id` generated via `crypto.randomUUID()`
2. The new copy's title is set to "{Original Title} (copy)"
3. The app transitions from shared view mode to full editor mode with the copied proposal loaded
4. The URL updates to the editor route (no share link in the URL after the copy is created — the copy is local until the user explicitly shares it)
5. No confirmation dialog is needed — `Edit as Copy` is not a destructive action

### Image Export (SHARE-03)

- Export captures the map stage DOM element using `html2canvas` or equivalent canvas-to-PNG approach
- Output: PNG file, filename is `{proposal-title-slug}.png` (title lowercased, spaces replaced with hyphens, special characters stripped)
- If title is "Untitled Proposal", filename is `ttc-proposal.png`
- Export resolution: device pixel ratio × the map stage dimensions at current viewport size (no forced upscaling)
- The toolbar, sidebar, and all chrome are excluded from the export region
- During export: the comparison banner (if visible) is hidden from the export region even if comparison mode is active
- The onboarding tooltip (if visible) is hidden from the export region

### Onboarding Tooltips (SHARE-07)

Shown only on first visit (tracked via `localStorage` flag `ttc-dreamer-onboarded`). A sequence of at most 3 tooltips:

| Step | Anchor | Tooltip heading | Tooltip body |
|------|--------|----------------|-------------|
| 1 | `Draw Line` toolbar button | Draw your first line | Click to start drawing a new transit line on the map. |
| 2 | `Add Station` toolbar button | Place stations | Click on any line to add a station along the route. |
| 3 | `Share` toolbar button | Share your proposal | Export an image or create a link to share your map. |

- Tooltips advance on `Got it` button click — next tooltip anchors to the next target button
- Any tooltip can be dismissed permanently with `Skip all` text link (14px weight 400, `var(--shell-dominant)` at 70% opacity, underline) which sets the `localStorage` flag and hides all remaining tooltips
- Tooltips are not shown if the user has been active for more than 60 seconds before the first tooltip would appear (they found their way around on their own)
- Tooltips do not appear in shared view mode
- Tooltip sequence starts after the map finishes loading (wait for `TorontoMap` to emit its ready event)

### Sidebar — Phase 5 Additions

The existing sidebar panel states from Phase 4 are unchanged. Phase 5 adds no new sidebar panel states. The sidebar is not shown in shared view mode.

### Toolbar — Phase 5 Additions

| Addition | Position | Behavior |
|----------|----------|----------|
| Map title inline field | Between tool group and baseline toggle | Editable label for `draft.title`; 14px weight 400, 200px max-width |
| `Share` button | Rightmost position, after `Add Line` | Opens share modal; `--shell-accent` background always |

All existing toolbar controls from Phases 1–4 are unchanged.

### Map Cursor States — Phase 5

All cursor states from Phases 3 and 4 are unchanged. No new cursor states are introduced in Phase 5. In shared view mode, the cursor is always `grab` (panning) / `grabbing` (active pan) — no editing cursors are ever shown.

| Onboarding step 3 body | Export an image or create a link to share your map. |
| Onboarding advance button | Got it |
| Onboarding dismiss all link | Skip all |
| Error: share link generation failed | Couldn't create a share link. Check your connection and try again. |
| Error: export failed | Couldn't export the image. Try again or resize the map and retry. |
| Error: shared proposal not found | This proposal isn't available. It may have been removed or the link is incorrect. |
| Empty shared view (proposal has no lines) | This proposal has no lines yet. |

Copy rules:
- Use imperative verbs for all action buttons: "Share", "Export", "Create", "Copy", "Edit"
- The `Edit as Copy` label uses title case — this is the only multi-word button that does so; keep it consistent
- Author name in shared view: if not set, show nothing — no "Anonymous" fallback in the view-mode banner
- "Copied!" feedback replaces the button label in place — no toast or notification
- All error messages name the problem first, then tell the user what to do (inherited from Phase 3 rule)
- Stats in the view-mode banner use `·` (middle dot, U+00B7) as the separator, not a dash or pipe
- The share link description uses "unlisted" not "private" or "secret" — consistent with product framing
- "(copy)" in the cloned proposal title uses lowercase with parentheses — do not capitalize
- `×` close buttons use the multiplication sign (U+00D7), not `x` or `X` (inherited from Phase 4 rule)
