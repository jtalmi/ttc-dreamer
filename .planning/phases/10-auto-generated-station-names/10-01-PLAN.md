    - reverseGeocode([lng, lat]) returns a Promise resolving to a street-based name string
    - When Nominatim returns `address.road` = "King Street West" and a cross-street exists in the response, result is "King St W & Spadina Ave" format (abbreviated common suffixes: Street->St, Avenue->Ave, Road->Rd, Drive->Dr, Boulevard->Blvd, West->W, East->E, North->N, South->S)
    - When Nominatim returns `address.road` but no cross-street, result is the road name alone (abbreviated)
    - When Nominatim returns no road, falls back to `address.suburb` or `address.neighbourhood`
    - When fetch fails (network error), returns null
    - When fetch returns non-200, returns null
    - When fetch times out (3s AbortSignal), returns null
    - Repeated calls with coordinates that round to the same 4-decimal key (e.g., 43.6532/-79.3832 and 43.6533/-79.3831) return the cached result without a second fetch
    - Requests include User-Agent header "TorontoTransitSandbox/1.0"
    - Zoom parameter is set to 17 for street-level precision
  </behavior>
  <action>
    Create lib/geocoding/reverse-geocode.ts:

    1. Define types:
       - NominatimAddress: { road?: string; suburb?: string; neighbourhood?: string; [key: string]: unknown }
       - NominatimResponse: { address: NominatimAddress; [key: string]: unknown }

    2. Create module-level cache: `const cache = new Map<string, string>()`
       - Cache key function: round lat/lng to 4 decimals, concatenate as "lat,lng"
       - 4 decimals = ~11m precision, appropriate for station snapping

    3. Create abbreviation helper: `abbreviateStreetName(name: string): string`
       - Replace common Toronto suffixes: Street->St, Avenue->Ave, Road->Rd, Drive->Dr, Boulevard->Blvd, Crescent->Cres, Court->Ct, Place->Pl, Trail->Trl, Way->Way (keep), Circle->Cir, Gate->Gate (keep), Lane->Ln, Terrace->Terr, Gardens->Gdns, Square->Sq, Park->Park (keep)
       - Replace directional suffixes: West->W, East->E, North->N, South->S
       - Case-sensitive word boundary replacements only (don't replace "Western" with "Wrn")

    4. Export async function `reverseGeocode(position: [number, number]): Promise<string | null>`:
       - Check cache first; return cached value if hit
       - Fetch `https://nominatim.openstreetmap.org/reverse?lat=${position[1]}&lon=${position[0]}&format=json&zoom=17` with:
         - Headers: { "User-Agent": "TorontoTransitSandbox/1.0" }
         - Signal: AbortSignal.timeout(3000)
       - Parse JSON response
       - Extract name: prefer `address.road`, then cross-street from response (Nominatim often includes nearby road in supplementary fields — check for second road name or use road alone), then `address.suburb`, then `address.neighbourhood`
       - Abbreviate the extracted name(s) with abbreviateStreetName
       - If intersection detected (two road names available), format as "Road1 & Road2"
       - Store in cache and return
       - On ANY error (fetch fail, timeout, parse error, missing address), return null

    5. Export `clearGeocodeCache()` for testing.

    Create tests/geocoding/reverse-geocode.test.ts:
    - Mock global.fetch using vi.fn()
    - Test: successful geocode with road returns abbreviated road name
    - Test: successful geocode with road + cross-street returns "Road1 & Road2"
    - Test: no road falls back to suburb
    - Test: no road or suburb falls back to neighbourhood
    - Test: network error returns null
    - Test: non-200 status returns null
    - Test: cache hit returns same result without second fetch call (assert fetch called once for two calls with nearby coords)
    - Test: abbreviation function correctly abbreviates "King Street West" to "King St W"
    - Call clearGeocodeCache() in beforeEach
