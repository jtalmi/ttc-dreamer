## Visual Hierarchy

- **Primary focal point:** the map canvas — inspectors and stats must never compete with the map for visual weight; they live entirely within the sidebar
- **Secondary focal point:** the active inspector panel header (line name with color stripe, or station name) and the live proposal stats summary row at the top of the sidebar
- **Tertiary focal point:** individual stat values, section dividers, and the before/after comparison toggle when active
- **Layer stacking order:** unchanged from Phase 3 — no new map canvas layers are introduced in Phase 4
- **Inspector constraint:** inspector panels open inside the existing 320px sidebar; they do not float over the map, they do not push the map, and they do not exceed the sidebar width
- **Comparison overlay:** when before/after mode is active, the map renders a subtle muted overlay on proposal elements — this is the only Phase 4 map canvas change
- **Accessibility rule:** all inspector controls and stat labels use visible text; minimum click target 44x44px for all inspector action buttons

## Typography

Four declared sizes — identical to Phases 1–3:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.3 |
| Heading | 20px | 600 | 1.25 |
| Display | 32px | 600 | 1.1 |

Phase 4 type assignments:

- Inspector panel heading (line name or station name): 20px, weight 600, line-height 1.25 (Heading role)
- Inspector section labels (e.g. "Type", "Length", "Stations"): 14px, weight 600, line-height 1.3 (Label role)
- Inspector value text (e.g. "Subway", "12.4 km", "7 stations"): 14px, weight 400, line-height 1.3 — Label size, Body weight
- Stat group heading (e.g. "Proposal", "Line"): 14px, weight 600, line-height 1.3 (Label role), muted at 70% opacity
- Stat value (e.g. "~32 min", "~$2.1B"): 16px, weight 600, line-height 1.5 (Body size, Label weight)
- Stat unit/qualifier (e.g. "estimated", "end-to-end"): 14px, weight 400, line-height 1.3 — Label size, Body weight, muted at 60% opacity
- Before/after toggle labels: 14px, weight 600 (Label role)
- Comparison delta label (e.g. "+3 stations"): 14px, weight 600, line-height 1.3 (Label role)

> Source: Phase 1 and Phase 3 UI-SPEC.md base typography. No new size roles introduced.

| Role | Value | Usage |
|------|-------|-------|

Accent reserved for: active inspector panel header indicator stripe, active before/after toggle button state, keyboard focus rings. No new accent uses beyond this list.

**Phase 4 inspector and comparison tokens (new additions to globals.css):**

| Token | Value | Usage |
|-------|-------|-------|
| `--inspector-header-stripe` | Matches the selected line's `color` property | Left border stripe on the line inspector header — uses the line's own color, not accent |
| `--inspector-divider` | `rgba(243, 238, 229, 0.12)` | Horizontal rule between inspector sections and stat groups |
| `--stat-value-muted` | `rgba(243, 238, 229, 0.60)` | Stat qualifiers and unit labels |
| `--stat-group-label` | `rgba(243, 238, 229, 0.70)` | Stat group headings ("Proposal", "Line") |
| `--comparison-baseline-overlay` | `rgba(24, 50, 74, 0.25)` | Semi-transparent veil applied to map canvas over proposal elements in before/after mode |
| `--comparison-delta-positive` | `#4CAF50` | Positive delta labels (e.g. "+3 stations added") |
| `--comparison-delta-neutral` | `rgba(243, 238, 229, 0.70)` | Neutral/no-change delta labels |

**Line inspector header rendering rules:**

- Left border stripe: 4px solid, color = the line's chosen color (not `--shell-accent`)
- Background: `var(--shell-secondary)` — matches sidebar chrome
- Heading: 20px weight 600, `var(--shell-dominant)` text

**Before/after comparison map behavior:**

- When "Before" is active: proposal line strokes are reduced to 40% opacity; proposal station dots are reduced to 40% opacity; the comparison toggle button uses `--shell-accent` background to indicate the override is live
- When "After" is active (default): proposal renders at full opacity, same as normal editing view
- Comparison does not affect baseline TTC or GO layers — they remain at full visibility in both states

## Interaction Contract

### Inspect Tool — Now Active (STATS-01, STATS-02)

The `Inspect` tool was scaffolded in Phase 3 and is now fully activated:

- Toolbar label: `Inspect` (unchanged from Phase 3 scaffold)
- When `Inspect` is the active tool: clicking a proposed line opens the **Line Inspector** panel in the sidebar
- When `Inspect` is the active tool: clicking a proposed station opens the **Station Inspector** panel in the sidebar
- Clicking elsewhere on the map (no proposal element hit) dismisses the active inspector and returns the sidebar to the **line list + stats** view
- Baseline TTC elements are not inspectable via the Inspect tool — clicking a TTC line or station while in Inspect mode does nothing and shows no error (consistent with Phase 3 delete behavior)
- Map cursor in Inspect mode: `pointer` (not `crosshair`)

### Line Inspector Panel (STATS-01)

Triggered by: clicking a proposed line with the Inspect tool active, or clicking a line row in the sidebar line list while in any mode.

Panel layout (top to bottom inside the 320px sidebar):

1. **Header strip:** 4px left border in the line's own color, line name (20px weight 600), mode badge (Subway / LRT / BRT at 14px weight 600, muted), and a close button (`×`, 44x44px tap target) in the top-right corner
2. **Divider** (`--inspector-divider`)
3. **Geometry section:** two rows — "Length" with computed km value, "Stations" with station count on this line
4. **Divider**
5. **Stats section:** four stat blocks in a 2×2 grid — Travel Time, Avg Stop Spacing, Est. Cost, Est. Ridership (see Stats section below for values and qualifiers)
6. **Divider**
7. **Connection section:** one row — "Interchanges" with count of shared/linked stations on this line

The inspector panel replaces the sidebar content area while open. The `Add Line` button is hidden while an inspector is open. The panel fits within the existing 320px sidebar width without scrolling for the typical case (up to ~10 stations). If content overflows, the panel body scrolls internally; the header strip stays pinned.

### Station Inspector Panel (STATS-02)

Triggered by: clicking a proposed station with the Inspect tool active.

Panel layout (top to bottom):

1. **Header strip:** station name (20px weight 600), and a close button (`×`, 44x44px tap target) in the top-right corner — no color stripe (stations inherit colors from their lines; the header stays neutral)
2. **Divider**
3. **Lines section:** lists each line this station belongs to as a row — 12px color dot + line name (14px weight 600). If the station is on only one line, a single row. If shared, all lines listed.
4. **Divider**
5. **Interchange section:** "Connected to baseline" row — if `linkedBaselineStationId` is set, shows the TTC station name; otherwise shows "None"
6. **Divider**
7. **Position section:** "Location" row — human-readable neighbourhood or cross-street hint derived from coordinates (nearest Toronto neighbourhood label from the existing context data). If no neighbourhood can be resolved: show coordinates rounded to 4 decimal places.

### Live Stats Sidebar (STATS-03, STATS-04, STATS-05)

When no inspector is open, the sidebar expanded view shows: the proposal line list (existing from Phase 3) **above** a new **stats summary section** pinned at the bottom of the sidebar content area.

Stats summary section layout:

- Section heading: "Proposal Stats" (14px weight 600, muted at 70% opacity)
- Divider
- Four primary stat blocks in a 2×2 grid:

| Stat Block | Label | Value format | Qualifier |
|------------|-------|-------------|-----------|
| Travel time | Travel Time | `~{N} min` | `end-to-end, longest line` |
| Avg stop spacing | Avg Spacing | `~{N} km` | `per line average` |
| Est. cost | Est. Cost | `~${N}B` or `~${N}M` | `rough order of magnitude` |
| Est. ridership | Est. Ridership | `~{N}K/day` | `directional estimate` |

- Divider
- Two secondary stat rows (single line each):

| Stat Row | Label | Value format |
|----------|-------|-------------|
| Station count | Stations | `{N} total` |
| Line length | Network | `{N} km total` |
| Interchange count | Interchanges | `{N} connections` |

All stat values use the `~` prefix (tilde) to signal directional/estimated nature. No stat is presented without this qualifier in the displayed value.

**Stats tone rule (locked from product constraint):** Stats are expressive and directional. No stat block may use the word "accurate", "official", "projected", or "planned". Qualifier copy must signal estimation ("rough order of magnitude", "directional estimate", "end-to-end"). This is enforced in the copywriting contract.

### Before/After Comparison Toggle (STATS-06)

A new control added to the top toolbar, positioned to the right of the existing baseline toggle (`Today` / `Future committed`) and to the left of the `Bus + Streetcar Corridors` button.

- Label when in "After" state (default): `Proposal View`
- Label when in "Before" state (comparison active): `Baseline View`
- Toggle is a single button that flips between the two states on click
- When "Before" (Baseline View) is active: the button uses `--shell-accent` background, white text; proposal elements on the map drop to 40% opacity; a banner strip appears at the bottom of the map canvas with the text "Comparing against baseline — click Proposal View to return"
- When "After" (Proposal View) is active (default): the button uses `--shell-secondary` background, white text (matches the inactive baseline toggle button style)
- The comparison toggle is only meaningful when the proposal has at least one line. When the proposal is empty, the toggle is present but visually muted (opacity 0.4) and non-interactive.
- The comparison does not change the baseline mode (`Today` / `Future committed`) — both toggles operate independently

### Sidebar Panel Navigation — Phase 4 Evolution

The sidebar `sidebarPanel` value from `EditorChromeState` now includes two new panel states:

| Panel State | Trigger | Shows |
|-------------|---------|-------|
| `list` | Default, no selection | Line list + stats summary section |
| `create` | Add Line button | Line creation panel (Phase 3, unchanged) |
| `drawing-status` | Active draw session | Drawing status strip (Phase 3, unchanged) |
| `inspect-line` | Click proposed line with Inspect tool | Line inspector panel |
| `inspect-station` | Click proposed station with Inspect tool | Station inspector panel |

Returning from any inspector panel to the list is triggered by: the `×` close button, pressing `Escape`, or clicking away on the map canvas with the Inspect tool.

### Map Cursor States — Phase 4 Addition

| Mode | Cursor |
|------|--------|
| `inspect` (hovering over a proposed line) | `pointer` |
| `inspect` (hovering over a proposed station) | `pointer` |
| `inspect` (hovering over map with no proposal element) | `default` |

All other cursor states from Phase 3 are unchanged.

| Stat value format — est. cost (millions) | ~${N}M |

| Stat value format — stations | {N} total |
| Stat value format — network | {N} km total |
| Stat value format — interchanges | {N} connections |
| Line inspector — geometry label "length" | Length |
| Line inspector — geometry label "stations" | Stations |

| Before/after toggle — default state label | Proposal View |
| Before/after toggle — comparison active label | Baseline View |

| Empty proposal stats (no lines yet) | Add a line to see proposal stats. |

| Error: stat computation fails | Stats unavailable. Check that the line has at least two waypoints. |

Copy rules:
- All stat values use the `~` tilde prefix — never omit it
- Qualifier text appears below or beside every primary stat value at reduced opacity
- Stat labels use sentence case, not all-caps
- Inspector close buttons use `×` (multiplication sign U+00D7), not `x` or `X`
- The words "accurate", "official", "projected", and "planned" are banned from all stat copy
- Safe-exit (cancel) buttons use contextual "Keep [thing]" labels (inherited from Phase 3 rule)
- Error messages name the problem first, then tell the user what to do (inherited from Phase 3 rule)

## Stat Formula Reference

These formulas define what values the stats section renders. They are inputs to the planner and executor — not design decisions — but they are recorded here to ensure copy format, precision, and qualifier tone are consistent.

| Stat | Formula | Display precision | Notes |
|------|---------|-------------------|-------|
| Travel time | `total_length_km / assumed_speed_kmh * 60` | Round to nearest minute | Speed: subway=60, lrt=35, brt=25 km/h. Uses the longest proposal line. |
| Avg stop spacing | `total_length_km / (station_count - 1)` | Round to 1 decimal km | Per-line average, then averaged across all lines. Zero stations → show "—". |
| Est. cost | `total_length_km * cost_per_km` | Round to 1 decimal; use B or M suffix | Cost rate: subway=$250M/km, lrt=$100M/km, brt=$30M/km. These rates are illustrative. |
| Est. ridership | `station_count * ridership_per_station_per_day` | Round to nearest 100; divide by 1000 for K | Rate: subway=2000/station/day, lrt=1200, brt=600. Directional only. |
| Network length | Sum of all line `waypoints` great-circle distances | Round to 1 decimal km | Uses Haversine or turf.js `length`. |
| Interchange count | Count of `ProposalStationDraft` entries with `lineIds.length > 1` or `linkedBaselineStationId` set | Integer | Counts both proposal-to-proposal and proposal-to-TTC links. |

**Tone enforcement:** Every displayed stat value is prefixed with `~` and accompanied by a qualifier string that uses the exact copy from the Copywriting Contract above. No formula value may be presented as definitive.
