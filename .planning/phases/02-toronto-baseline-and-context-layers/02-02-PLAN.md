Output: Context label layers rendering inside the TorontoMap component, with supporting GeoJSON data files for neighbourhoods, landmarks, and major streets.
</objective>

    3. **Station hover tooltip** — Use react-map-gl `Popup` component:
       - Track `hoverStation` state: `{ name: string; lng: number; lat: number; type: "ttc" | "go" } | null`
       - On the `<Map>` component (in TorontoMap), use `onMouseMove` to detect hover over `ttc-stations` circle layer (from Plan 01) and `go-stations` circle layer
       - When hovering a TTC station, show Popup with `"{Station Name} -- Line {N}"` (per UI-SPEC copywriting: use an em-dash)
       - When hovering a GO station, show Popup with `"{Station Name} -- GO"` (per UI-SPEC)
       - On `onMouseLeave`, clear `hoverStation`
       - Popup: `anchor="bottom"`, `closeButton={false}`, `closeOnClick={false}`, `offset={10}`
       - Style the popup text at 14px, font-family var(--font-sans) — but note that maplibre-gl.css controls Popup chrome; keep it minimal

    Export `StationLabels` as a named export. The tooltip state management and `onMouseMove`/`onMouseLeave` handlers will need to live in `toronto-map.tsx` since they require Map-level event binding.

    **Step 2 — Wire all label layers into TorontoMap (`components/editor/toronto-map.tsx`):**
    Modify the existing TorontoMap component to:
    - Add data loading for `neighbourhoods.geojson`, `landmarks.geojson`, and `major-streets.geojson` in the existing useEffect (add three more fetch calls alongside the existing four)
    - Create loader functions in `lib/baseline/baseline-data.ts` for the three new files: `loadNeighbourhoods()`, `loadLandmarks()`, `loadMajorStreets()`. Add them to the barrel export.

    - Add `interactiveLayerIds={["ttc-stations"]}` to the `<Map>` component (only TTC stations are interactive in Phase 2; GO is context-only)
    - Add `onMouseMove` and `onMouseLeave` handlers for station tooltip:
      - `onMouseMove`: check `e.features` for hits on `ttc-stations` layer; if found, set hover state with station name, coordinates, and type
      - `onMouseLeave`: clear hover state
      - For GO stations: also check `go-stations` layer in `onMouseMove`. Even though GO pointer-events are disabled in the UI-SPEC, the tooltip is permitted per the copywriting contract. If this contradicts the "pointer-events disabled" rule, then SKIP go station hover — only show TTC hover tooltips.
    - Render in correct stacking order (bottom to top per UI-SPEC):
      1. `<ContextLabels>` (streets at bottom, then neighbourhoods, then landmarks)

      4. `<StationLabels>` (TTC + GO station name text labels — rendered ABOVE dots)
      5. Popup (tooltip — rendered last, floats above everything)

    **Note on stacking:** The actual stacking order in MapLibre is determined by the order layers are added to the style. react-map-gl adds layers in component render order. So the render order of Source+Layer children inside `<Map>` must follow the UI-SPEC stacking specification.
