    **Step 1 — Install dependencies:**
    ```bash
    npm install react-map-gl maplibre-gl
    ```
    Verify react-map-gl@8.x and maplibre-gl@5.x are installed (see RESEARCH.md for exact confirmed versions).

    **Step 2 — Fetch TTC and GO station data from City of Toronto ArcGIS:**
    ```bash
    mkdir -p public/data

    # TTC subway routes (Layer 11 — all routes)
    curl -s "https://gis.toronto.ca/arcgis/rest/services/cot_geospatial7/FeatureServer/11/query?where=1%3D1&outFields=*&f=geojson" \
      > public/data/ttc-routes.geojson

    # TTC subway stations (Layer 8, filtered to Subway Stations)
    curl -s "https://gis.toronto.ca/arcgis/rest/services/cot_geospatial7/FeatureServer/8/query?where=PT_TYPE%3D%27Subway%20Stations%27&outFields=*&f=geojson" \
      > public/data/ttc-stations.geojson

    # GO Train stops (Layer 7)
    curl -s "https://gis.toronto.ca/arcgis/rest/services/cot_geospatial7/FeatureServer/7/query?where=1%3D1&outFields=*&f=geojson" \
      > public/data/go-stations.geojson
    ```
    After fetching, validate each file is valid JSON with `features` array. If any curl fails (network issues), create minimal hand-authored GeoJSON with at least 10 representative features so the build does not block.

    **Step 3 — Hand-author GO route lines:**
    Create `public/data/go-routes.geojson` as a hand-authored FeatureCollection with ~7 GO rail corridors as simplified LineString features connecting the confirmed GO station points. Use the GO station coordinates as anchors. Corridors to include:
    - Lakeshore West (Union → Long Branch area)
    - Lakeshore East (Union → Rouge Hill area)
    - Barrie (Union → Downsview Park area)
    - Stouffville (Union → Scarborough/Markham area)
    - Kitchener (Union → Weston/Etobicoke area)
    - Richmond Hill (Union → Oriole area)
    - Milton (Union → Kipling area)
    Each line needs only 3-6 waypoints — these are context corridors at 75% opacity, not precise geometry.

    **Step 4 — Add map layer color tokens to globals.css:**

    ```css
    --station-dot-ttc: #FFFFFF;
    --station-dot-go: #FFFFFF;
    ```

    **Step 5 — Create baseline type definitions:**
    Create `lib/baseline/baseline-types.ts` with TypeScript types for the GeoJSON feature properties used in TTC routes, TTC stations, GO stations, and GO routes. These types should reflect the actual ArcGIS field names (e.g., `ROUTE_ID`, `PLACE_NAME`, `PT_TYPE`). Export all types as named exports. Use `import type` for GeoJSON types from the `geojson` module.

    Create `lib/baseline/baseline-data.ts` with async loader functions:
    - `loadTtcRoutes(): Promise<FeatureCollection>` — fetches `/data/ttc-routes.geojson`
    - `loadTtcStations(): Promise<FeatureCollection>` — fetches `/data/ttc-stations.geojson`
    - `loadGoRoutes(): Promise<FeatureCollection>` — fetches `/data/go-routes.geojson`
    - `loadGoStations(): Promise<FeatureCollection>` — fetches `/data/go-stations.geojson`
    Each function fetches from the public directory and returns parsed JSON. Include a `TORONTO_VIEW` constant: `{ longitude: -79.387, latitude: 43.653, zoom: 11 }`.

    Create `lib/baseline/index.ts` as a barrel re-exporting all types and functions from the two files above.

    **Step 1 — Create TTC layer component (`components/map/ttc-layers.tsx`):**
    A `"use client"` component that receives a `routes: FeatureCollection` and `stations: FeatureCollection` prop. Renders:
    - A `<Source>` for TTC routes with `<Layer>` children for each line, using `filter: ["==", ["get", "ROUTE_ID"], N]` to separate lines. Paint properties per UI-SPEC:
      - Line 1 (Yonge-University): `line-color: "#FFC41F"`, `line-width: 4`
      - Line 2 (Bloor-Danforth): `line-color: "#009A44"`, `line-width: 4`
      - Line 4 (Sheppard): `line-color: "#800080"`, `line-width: 4`
      - Line 5 (Eglinton): `line-color: "#DF6C2B"`, `line-width: 4`
      - All lines: `line-join: "round"`, `line-cap: "round"`
    - A `<Source>` for TTC stations with a `<Layer>` of type `circle`:
      - `circle-radius: 4` (8px diameter), `circle-color: "#FFFFFF"`, `circle-stroke-width: 2`
      - `circle-stroke-color`: use a data-driven expression matching ROUTE_ID to the line color. If the station data does not include ROUTE_ID, use `"#18324A"` as the default stroke color.
    - Import `Source, Layer` from `react-map-gl/maplibre` and type `LineLayer, CircleLayer` from `react-map-gl/maplibre`.
    - Export as named export `TtcLayers`.

    **Step 2 — Create GO layer component (`components/map/go-layers.tsx`):**
    A `"use client"` component that receives `routes: FeatureCollection` and `stations: FeatureCollection` props. Renders:
    - GO route lines: `line-color: "#007A3D"`, `line-width: 3`, `line-dasharray: [6, 3]`, `line-opacity: 0.75`
    - GO station circles: `circle-radius: 3.5` (7px diameter), `circle-color: "#FFFFFF"`, `circle-stroke-width: 2`, `circle-stroke-color: "#007A3D"`
    - Export as named export `GoLayers`.
    - GO layers are NOT added to `interactiveLayerIds` (making them non-interactive per MAP-02).

    **Step 3 — Create TorontoMap component (`components/editor/toronto-map.tsx`):**
    A `"use client"` component. This is the main map.
    - Import `Map` from `react-map-gl/maplibre` and `"maplibre-gl/dist/maplibre-gl.css"`.
    - Import `TtcLayers` and `GoLayers`.
    - Import baseline data loaders from `@/lib/baseline`.
    - Use `useState` + `useEffect` to load all four GeoJSON files on mount.
    - Show `"Loading Toronto map..."` text centered in a div while data loads (per UI-SPEC copywriting).
    - Show `"Couldn't load the map. Check your connection and refresh the page."` if fetch fails (per UI-SPEC).
    - Once loaded, render `<Map>` with:
      - `initialViewState={TORONTO_VIEW}` (from baseline data)
      - `mapStyle` using `process.env.NEXT_PUBLIC_MAPTILER_KEY` with dataviz-light style: ``https://api.maptiler.com/maps/dataviz-light/style.json?key=${process.env.NEXT_PUBLIC_MAPTILER_KEY}``
      - `style={{ width: "100%", height: "100%" }}`
      - `attributionControl={{ compact: true }}`
    - Render children in correct stacking order per UI-SPEC (bottom to top): GO lines → GO stations → TTC lines → TTC stations. This means `<GoLayers>` renders BEFORE `<TtcLayers>` inside the `<Map>`.
    - If `NEXT_PUBLIC_MAPTILER_KEY` is not set, log a console.warn with a helpful message and still attempt to render (the map will show a blank canvas with overlay layers).
    - Export as default export.
