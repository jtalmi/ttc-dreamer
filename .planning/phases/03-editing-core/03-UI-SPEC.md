**Proposed line rendering rules:**

- Proposed line stroke width: 4px (same as TTC lines) â€” proposal lines must compete visually on equal footing with the baseline
- Proposed line stroke: solid, fully opaque when committed; `--proposal-line-pending` opacity while actively drawing
- Proposed station dot: 8px diameter white circle (`--proposal-station-dot`), 2px border in the line's chosen color â€” matches TTC station dot pattern exactly
- Selected proposed element: renders `--selected-element` as a 2px glow/halo at 8px blur â€” does not change the element's own stroke color
- Snapping cue ring: 16px diameter ring centered on snap target, `--snap-cue-ring`, 1.5px stroke, no fill
- In-progress drawing cursor dot: 8px filled circle in `--drawing-cursor-dot`, centered on map cursor position

**Default color palette for new lines (sequential assignment when user does not pick):**

## Interaction Contract

### Toolbar â€” Tool Modes (Phase 3 activations)

The existing toolbar from Phase 1 carries forward. Phase 3 activates the previously scaffold-only tools with real behavior:

| Tool Label | Mode Key | Phase 3 Behavior |
|------------|----------|------------------|
| `Select` | `select` | Click a proposed line or station to select it; drag to move; shows selection highlight |
| `Draw Line` | `draw-line` | Click to place route waypoints; double-click to finish; draws in-progress segment from last waypoint to cursor |
| `Add Station` | `add-station` | Click on a proposed line segment to place a station at that point on the segment; triggers snapping and interchange suggestions |
| `Inspect` | `inspect` | Scaffolded â€” not fully active in Phase 3; clicking shows no-op (inspector lands in Phase 4) |

- The active tool pill uses `--shell-accent` background, white text, as established in Phase 1.
- Keyboard shortcut labels are not shown in Phase 3 (Phase 5 polish).

### New Line Creation Flow

1. User presses the `Add Line` button in the sidebar (or via toolbar context â€” see Sidebar section).
2. A **line creation panel** slides open inside the sidebar (320px wide panel).
3. Panel contains: line name field (prefilled with a generated default, e.g. "Line A"), mode selector (Subway / LRT / BRT), and a color picker (swatch grid).
4. User fills in name, selects mode, optionally picks color; presses `Start Drawing`.
5. Toolbar automatically switches to `Draw Line` tool and the line creation panel collapses to a **drawing status strip** (showing the line name and a `Finish Line` button).
6. User clicks waypoints on the map to build the route. The in-progress segment follows the cursor in `--proposal-line-pending` color.
7. User presses `Finish Line` (in the strip) or double-clicks on the map to commit.
8. The committed line appears at full opacity with its chosen color.

### Extend Existing Line Flow

1. User activates `Draw Line` tool and clicks on an endpoint of an existing TTC or proposal line.
2. The app recognizes the endpoint hit and enters **extend mode**: a visual indicator (2px dashed ring, `--snap-cue-ring`) appears on the endpoint.
3. User clicks waypoints to draw the extension from the endpoint.
4. Finish by double-click or `Finish Line` strip button.
5. The extension is attached to the existing line and takes that line's color.

### Branch Existing Line Flow

1. User activates `Draw Line` tool and clicks on an existing TTC or proposal line segment (not an endpoint).
2. The app enters **branch mode**: a 2px dashed ring appears at the branch point.
3. User draws from the branch point. The new branch is a separate line entity in the proposal (not merged into the original line).
4. Branch lines use the next default color in the palette sequence, distinct from the parent line.
5. User can rename the branch in the line creation panel.

### Station Placement

- `Add Station` tool: click on a proposed line segment to place a station at that point on the segment.
- The station snaps to the nearest point on the segment (snapping tolerance: 12px at current zoom).
- If the placement point is within 20px screen-distance of an existing station (TTC or proposal), the **interchange suggestion badge** appears (see below).
- Placed stations immediately show a dot and label.

### Shared Station / Interchange Suggestion

- When a new station placement is near an existing station, a badge appears next to the candidate station location.
- Badge copy: `Make interchange?` with two action buttons: `Yes` and `No`.
- `Yes` links the new station to the existing station as a shared station entity.
- `No` places the new station as a fully separate entity; the two stations remain visually close but structurally independent.
- Crossing lines that do not go through the interchange flow are never auto-connected. There is no implicit connection from line crossings.
- Badge auto-dismisses after 8 seconds if no action is taken; defaults to `No` on dismiss.

### Snapping Cues

- When the cursor is within 12px screen-distance of a snappable point (existing station, endpoint, or segment), the snapping ring (`--snap-cue-ring`, 16px diameter) appears centered on the snap target.
- Snapping is never forced: the user can ignore the cue and place freely.
- Snapping cue appears and disappears instantly â€” no animation delay.

### Select and Move

- `Select` tool: single click on a proposed line highlights it with `--selected-element` glow. Click on a proposed station highlights it.
- Drag a selected proposed station to reposition it.
- Drag a selected waypoint on a proposed line route to reshape the route at that point.
- Baseline TTC stations and lines are not selectable or movable.

### Line Naming and Coloring

- Line name is set during the line creation panel flow.
- After creation, clicking a selected line in the sidebar panel opens an **inline edit** for the name field (single-click on the label in the sidebar).
- Color picker: a grid of 12 swatches (8px gap between swatches). Swatches are 28x28px circles. The active color has a 2px `--shell-accent` outline ring at 2px offset. Custom hex input is not provided in Phase 3.
- Swatch options: the 5 default palette colors plus 7 additional curated colors (avoiding all TTC canonical colors). Total 12 swatches.

### Station Naming

- When a station is placed, a **station name popover** appears immediately (attached to the station dot, above it by 8px).
- Popover contains: a single text field prefilled with "Station" + sequential number (e.g. "Station 1"), and a `Save Name` button.
- User can accept the default or type a custom name. Press Enter or click `Save Name` to confirm.
- Popover closes on confirm. The station label updates on the map.
- If user clicks away without confirming, the name defaults to the prefilled value.

### Undo / Redo / Delete

- Keyboard: `Cmd+Z` / `Ctrl+Z` for undo; `Cmd+Shift+Z` / `Ctrl+Shift+Z` for redo.
- Toolbar: no undo/redo buttons in Phase 3 toolbar (keyboard only; button controls land in Phase 5 polish).
- Delete: `Backspace` or `Delete` key when a proposed element is selected removes it.
- Deleting a proposed line triggers a **confirmation dialog** (see Copywriting Contract).
- Deleting a proposed station that is an interchange: confirmation dialog warns that the interchange link will also be removed.
- Baseline TTC elements are not deletable â€” pressing Delete while a TTC element appears selected does nothing and shows no error.

### Sidebar â€” Phase 3 Evolution

The sidebar shell (320px expanded, 64px collapsed rail) is now active content:

- **Collapsed state:** rail shows only the `â€ş` expand affordance (unchanged from Phase 2).
- **Expanded state â€” default:** shows the proposal line list. Each line is a row with: a 12px color swatch dot, the line name (14px Label weight), and a mode badge (Subway / LRT / BRT, 14px Label, muted).
- **`Add Line` button:** pinned at the bottom of the line list, full width, `--shell-accent` background, 14px Label weight, label `Add Line`.
- **Line list row hover:** row background changes to `rgba(243, 238, 229, 0.08)` â€” subtle, does not use accent.
- **Active drawing status strip:** when in Draw Line mode, the strip replaces the top area of the sidebar with: line name (14px Label, bold), drawing instructions ("Click to place waypoints. Double-click to finish."), and a `Finish Line` button (`--shell-accent` background).
- **Line creation panel:** slides in from the sidebar when `Add Line` is pressed â€” full sidebar width, pushes or replaces the line list while open. Contains: name field, mode selector, color swatch grid, and `Start Drawing` button.

Copy rules:
- Use imperative verbs for all action buttons: "Add", "Start", "Finish", "Save", "Delete", "Clear"
- Safe-exit (cancel) buttons use contextual "Keep [thing]" labels â€” never the bare word "Cancel"
- Destructive confirm buttons name the specific action and target: "Delete Line", "Delete Station", "Clear Proposal"
- Line names appear in quotation marks in confirmation copy
- Station count in shared station warning uses the exact number of lines sharing the station
- "This cannot be undone" appears in every destructive confirmation
- Error messages name the problem first, then tell the user what to do
