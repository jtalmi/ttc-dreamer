**2. Extend proposal-state.ts** with new action types and reducer cases:

New actions (add to `EditorShellAction` union):
- `startDrawing`: payload `{ lineId: string; mode: "new" | "extend" | "branch"; initialWaypoint?: [number, number] }` — sets `drawingSession` in chrome, switches tool to `draw-line`, sets `sidebarPanel` to `"drawing-status"`.
- `addWaypoint`: payload `[number, number]` — appends to `drawingSession.waypoints`.
- `updateCursorPosition`: payload `[number, number] | null` — updates `drawingSession.cursorPosition` for ghost segment.
- `finishDrawing`: no payload — commits `drawingSession.waypoints` to the line's `waypoints`, clears `drawingSession`, switches tool back to `select`, sets `sidebarPanel` to `"list"`.
- `cancelDrawing`: no payload — clears `drawingSession` without committing, removes the line if it has no committed waypoints, sets `sidebarPanel` to `"list"`.
- `addLine`: payload `{ name: string; mode: TransitMode; color: string }` — creates a new `ProposalLineDraft` with a generated UUID, empty waypoints, pushes to `draft.lines`. Does NOT automatically start drawing (that is a separate action).
- `setSidebarPanel`: payload `"list" | "create" | "drawing-status"` — updates `chrome.sidebarPanel`.
- `setSelectedElement`: payload `string | null` — updates `chrome.selectedElementId`.

Ensure `setActiveTool` case also clears `pendingInterchangeSuggestion: null` (per RESEARCH pitfall 4). Ensure `setActiveTool` cancels any active `drawingSession` if switching away from `draw-line` (dispatch `cancelDrawing` logic inline).

For UUID generation, use `crypto.randomUUID()` (available in modern browsers and Node 19+).

Baseline protection (EDIT-05): No action in the reducer mutates any TTC baseline data. TTC data is loaded separately and never enters the proposal draft.

**4. Create proposal-history.ts** (new file) per RESEARCH Pattern 4:

- `HistoryState` type: `{ past: ProposalDraft[]; present: EditorShellState; future: ProposalDraft[] }`.
- `HistoryAction` type: `{ type: "undo" } | { type: "redo" } | EditorShellAction`.
- `HISTORY_ACTIONS` Set: actions that push to history — `"addLine"`, `"finishDrawing"`, `"addWaypoint"` (no — only final commits should push), correction: push history on `"addLine"`, `"finishDrawing"`, `"placeStation"`, `"deleteStation"`, `"deleteLine"`, `"linkInterchange"`, `"updateLineName"`, `"updateLineColor"`, `"updateStationName"`, `"moveStation"`, `"moveWaypoint"`. Do NOT push on `"addWaypoint"`, `"updateCursorPosition"`, `"startDrawing"`, `"cancelDrawing"` or any chrome action.
- `historyReducer(state: HistoryState, action: HistoryAction): HistoryState`:
  - `"undo"`: if `past.length === 0` return state. Pop last from `past`, set as `present.draft`, push current `present.draft` to `future`. Keep `chrome` unchanged.
  - `"redo"`: if `future.length === 0` return state. Pop first from `future`, set as `present.draft`, push current `present.draft` to `past`. Keep `chrome` unchanged.
  - Otherwise: delegate to `proposalEditorReducer`. If action type is in `HISTORY_ACTIONS`, push current `present.draft` to `past` (max 50 entries — slice), clear `future`. Otherwise just update `present`.
- `createInitialHistoryState(): HistoryState` — wraps `createInitialProposalDraft()`.
